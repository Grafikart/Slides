Comment rejoindre une équipe ?

Problématiques

  • Comprendre l'équipe
  • Comprendre les process
  • Comprendre le code

Étape 1

Onboarding

Présentation de l'équipe

  • Présentation de collaborateurs
  • Présentation des personnes clefs
  • Organigramme

Réunion métier

  • On ne parle pas de code
  • Comprendre l'objectif du produit
  • Comprendre l'existant

Réunion technique

  • Les outils
  • Le code
  • Mise en place du projet

Réunion process

  • Processus de travail
  • Outils de communication
  • Configuration technique (VPN, Clefs...)

Étape 2

Découverte

Participation au "daily"

  • Découverte de l'équipe
  • Découverte du processus

Premières tâches

  • Tâches simples et isolées
  • Pair programming

Revue de code

  • Obtenir des critiques pour s'améliorer
  • Comprendre

Étape 3

Intégration

Vous faites partie de l'équipe

Les pièges à éviter

Être trop "timide"

Tout remettre en question

  • Respecter les conventions
  • L'équipe c'est les compromis
  • Proposer mais pas imposer

Ne pas être susceptible

  • Une revue de code n'est pas une attaque
  • On peut se tromper (mais pas trop)

Soyez constructif

  • Ce n'est pas un concours
  • Collaboration et pas compétition

Bon courage !
